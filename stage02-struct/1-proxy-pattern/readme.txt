1. 静态代理:
1.1 好处是简单易实现、编译期校验早、运行性能优。
1.2 缺陷是代码冗余、维护成本高、灵活性差，无法适配多目标类和动态扩展场景。
1.3 例如，当房东数量扩展到一百个，如果使用静态代理，会导致类爆炸，并且冗余很高。

2. 动态代理
2.1动态代理通过统一处理器在运行时动态生成代理对象，能够灵活代理多个接口实现类，避免代码冗余和维护成本。
2.2但其基于反射调用性能略低，且只能代理接口无法代理类。


3. 对比
  特性       静态代理    	      动态代理
实现时机	     编译时	           运行时
代理对象	    手动编写	          自动生成
代码量	   多（类爆炸）	     少（通用处理器）
性能       高（直接调用）	     中（反射调用）
灵活性	        低	             高
适用场景	 代理类少、方法固定	代理类多、需要扩展
